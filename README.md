# IA Segura para Iniciantes: Prática Ágil com VSCode, GitHub e Mentalidade de Cibersegurança

Repositório do projeto: https://github.com/iranyassis/kensei

## 1. Descrição do Projeto
Este projeto acadêmico demonstra como estudantes iniciantes podem construir um fluxo de trabalho real com Visual Studio Code, GitHub e ferramentas de Inteligência Artificial. O objetivo principal é consolidar conceitos de desenvolvimento, integração de IA e segurança desde o primeiro dia.

## 2. Objetivos de Aprendizagem
- Entender o ciclo básico de desenvolvimento em VSCode (criar, editar, commit, push).
- Dominar controle de versão com Git e GitHub.
- Aplicar IA para gerar código inicial, revisar código e sugerir melhorias de segurança.
- Desenvolver pensamento crítico para validar resultados de IA.
- Trabalhar mentalidade de segurança (Secure by Design).

## 3. Tecnologias Utilizadas
- Visual Studio Code
- Git
- GitHub
- Ferramentas de IA (ChatGPT / Copilot / extensões AI)
- Markdown (`README.md`)
- Terminal integrado / PowerShell / WSL

## 4. Etapas Realizadas
1. Configuração inicial: instalar VSCode, extensões Git/IA e criar repositório no GitHub.
2. Inicialização do projeto: `git init`, `code .`, criar branch `main` e arquivo `README.md`.
3. Desenvolvimento guiado: escrever conteúdo em Markdown e solicitar à IA exemplos de comandos, abordagens e boas práticas.
4. Controle de versão: `git add .`, `git commit -m "inicial: estrutura do projeto"`, `git push origin main`.
5. Revisão com IA: pedir sugestão de melhorias de segurança, aplicar correções e refazer commits.
6. Documentação final: compilar seções obrigatórias e inserir seção de prompt e assinatura.

Exemplo prático de comando:
- `git status`
- `git checkout -b feature/seguranca`
- `git merge --no-ff feature/seguranca`

## 5. Aplicação de IA no Projeto
- Geração de esqueleto de README e sugestões de seção.
- Exemplos de código e scripts de segurança.
- Checklist automático de configuração segura (validação de dados, tratamento de erros, uso de variáveis de ambiente).
- Revisão de texto e clareza (legibilidade para público iniciante).

Por que é importante:
- A IA acelera aprendizado e reduz tempo de busca.
- Ajuda o iniciante a se concentrar na lógica e no entendimento.
- Fornece consultoria e feedback imediato.

## 6. Boas Práticas de Segurança
- Nunca commit dados sensíveis (senhas, chaves, tokens). Use `.gitignore`.
- Configurar autenticação forte no GitHub (2FA).
- Usar branches para isolar mudanças e revisar PRs.
- Revalidar conteúdo de IA (não copiar sem entender).
- Monitorar dependências e evitar pacotes maliciosos.
- Ferramentas simples:
  - `git log --oneline`
  - `git diff`
  - `gh secret set <NOME_VARIAVEL>`

Exemplo de `.gitignore`:
- `*.env`
- `node_modules/`
- `*.key`

## 7. Aprendizados Obtidos
- Evolução técnica: dominar workflow Git + GitHub em um ciclo completo, usar VSCode como IDE central e integrar comandos de IA com foco em entrega.
- Evolução mental: pensamento crítico ao avaliar respostas automáticas, postura “segurança desde o início” e confiança para avançar em projetos reais.

## 8. Conclusão
Este projeto confirma que iniciantes podem construir um fluxo profissional com IA, VSCode, Git e segurança. A jornada é prática, incremental e inspiradora: cada commit conta, cada dúvida vira aprendizagem.

> A área de tecnologia é uma estrada de crescimento constante. Continue testando, revisando e compartilhando conhecimento.
